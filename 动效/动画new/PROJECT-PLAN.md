## 已锁定决策

### 动画技术路线

- **路径形变（SVG Path Morphing）**：用 flubber.js 做相邻帧之间的路径插值
- **底部锚定**：所有帧共享底部基线（Figma 验证：5 帧底部 Y ≈ 199），向上生长
- **统一坐标系**：以 Frame 5 的 107×133 为基准空间，其他帧通过 Y 偏移底部对齐

### 5 帧关键帧数据（从 Figma 采集）

| 帧 | 名称 | SVG 尺寸 | Y 偏移(底部对齐) | 元素 |
|----|------|----------|-----------------|------|
| 1 | 小堆堆 | 43×13 | +120 | 土堆 |
| 2 | 两叶枝干 | 65×92 | +41 | 枝干(Y形) + 左叶 + 右叶 |
| 3 | 分叉长大 | 107×105 | +28 | 枝干(更宽分叉) + 左叶 + 右叶 |
| 4 | 新芽抽出 | 107×105 | +28 | 枝干 + 左叶 + 右叶 + 新芽枝干 |
| 5 | 花骨朵 | 107×133 | 0 | 枝干 + 叶×2 + 新芽(更长) + 花 |

### 4 段过渡动画方案

| 过渡 | 核心效果 | 技术手段 |
|------|---------|---------|
| 1→2 | 土堆膨胀 → Y 形枝干 + 叶子涌出 | 土堆→枝干 flubber morph；叶子 scale(0→1) + opacity |
| 2→3 | 枝干分叉向两侧撑开 | 枝干 path morph（Y 形→宽分叉）；叶子跟随分叉尖端位移 |
| 3→4 | 分叉中间向上抽出新芽 | 枝干不变；新芽从 0 高度参数化生长（已验证：test-sprout-grow.html） |
| 4→5 | 整体拔高 + 顶端花骨朵 | 所有元素 path morph 到更大版本；花从 scale(0) 出现 |

### 不做什么

- 不做 3D
- 不做视频渲染
- 不重绘用户的 SVG（直接复用导出路径）

## 约束

- 技术：单 HTML 文件，file:// 可直接打开，依赖 flubber.js (CDN)
- 素材：直接使用 `1/` ~ `5/` 文件夹中的 SVG 路径数据
- 精度：元素位置和尺寸必须与 Figma 精确匹配
