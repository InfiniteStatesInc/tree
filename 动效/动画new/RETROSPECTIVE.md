### [2026-04-23] - 参数化路径插值（Parametric Path Interpolation）实战经验

**场景**：祈祷树 stage 2→3 分叉动画，需要从一根树干裂开为左右两枝，中间态造型必须自然、和设计稿终态一致。

**学到的**：

- **技术选型**：相邻帧结构相似时用参数化控制点插值（逐点 lerp），形状差异大时用 flubber（自动配对）。本项目 1→2 用 flubber，2→3 用参数化插值。
- **退化段技巧**：源帧 3 段 path、目标帧 5 段 path 时，在源帧关键点（tip）插入 2 个退化段（所有控制点重合于一点），使两帧段数对齐后逐点 lerp，渲染结果和原 path 完全一致。
- **终值必须从设计稿提取**：手编"看起来差不多"的控制点坐标会导致枝干过粗、造型变形，且后续帧连锁变形无法延续。正确做法是从实际 SVG path 逐个提取贝塞尔控制点坐标。
- **附件元素时间函数必须精确匹配锚点**：叶子的 lerp timing 和枝尖的 stagger timing 用了不同的数学表达式（`easeInOutCubic(min(1, raw*1.18))` vs `min(1, easeInOutCubic(raw)*1.18)`），虽然看似等价但实际曲线不同，导致动画中间一瞬间叶子和枝尖脱节。修复方法：直接复制锚点的时间表达式，不做"等价改写"。
- **固定视角**：两帧底部中心不同时（stage 2 中心 x=39，stage 3 中心 x=72），需要对目标帧所有 x 坐标统一偏移（-33），否则动画过程中画面整体漂移。

**做对了的：**

| 决策 | 为什么对了 | 证据 |
|------|----------|------|
| 退化段对齐结构 | 让 lerp 可行且无跳变 | 用户确认"展开方式完全没问题" |
| 从设计稿提取终值 | 保证终态和设计一致 | 用户确认"走势和粗细都对了" |
| 底部偏移固定视角 | 动画不再漂移 | 用户一次通过 |

**做错了的：**

| 决策 | 哪里错了 | 根因 | 修了几轮 |
|------|---------|------|---------|
| 第一版手编控制点 | 枝干过粗，造型不匹配 | 没读设计稿 SVG 数据就动手 | 2轮 |
| 叶子时间函数"等价改写" | 中间态叶子脱节 | 数学直觉错误，easing 和乘法不可交换 | 2轮 |
| 第一版强制对称分叉 | 和设计稿不一致（设计是非对称的） | 假设植物分叉应该对称，没看设计 | 1轮 |

**下次要注意**：
- 有设计稿 SVG 时，第一步提取所有 path 控制点坐标作为 lerp 终值
- 附件元素时间函数：直接引用锚点变量（如 `tTop`），不做数学变换
- 不假设对称——先看设计稿实际数据
